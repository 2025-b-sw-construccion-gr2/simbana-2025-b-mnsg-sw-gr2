<h1>Prompt utilizado:</h1>

Necesito que me ayudes a desarrollar un caso de estudio para mi trabajo de Gestión de Configuración de Software (SCM). El caso debe tratar sobre el incidente ocurrido en GitHub en octubre de 2018, cuando la plataforma presentó una caída global que afectó la disponibilidad de repositorios, acciones y servicios durante varias horas, generando retrasos en proyectos y despliegues a nivel mundial.

Quiero que sigas esta estructura:

- Resumen del caso: explicar qué ocurrió, con fuente confiable y enlace.
- Problema identificado: qué causó la caída o qué vulnerabilidad o falla se presentó.
- Clasificación del mantenimiento: según los tipos (correctivo, adaptativo, perfectivo o preventivo), justificar cuál aplica.
- Procesos SCM involucrados: cómo ayudaron el control de versiones y la gestión de cambios a resolver el problema.
- Impacto en el ciclo de vida del software (SDLC): qué fases se vieron afectadas y cómo.
- Beneficios del SCM: qué permitió resolver el problema de forma más eficiente.
