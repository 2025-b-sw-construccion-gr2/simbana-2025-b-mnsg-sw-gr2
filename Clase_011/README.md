<h1>Prompt utilizado:</h1>

Quiero que actúes como un estudiante de Ingeniería de Software que debe realizar un informe breve (máximo dos páginas) sobre ingeniería inversa y sus implicaciones legales.

El objetivo es analizar dos aplicaciones comerciales ampliamente utilizadas: Visual Studio Code y IntelliJ IDEA.

El informe debe incluir lo siguiente:

- Nombre, versión y empresa de cada aplicación.
- Un resumen de los hallazgos legales obtenidos de sus licencias o términos de uso (centrado en ingeniería inversa, descompilación, modificación e interoperabilidad).
- Un análisis comparativo que responda estas preguntas:
    1. ¿La licencia permite la ingeniería inversa? ¿Total o parcialmente?
    2. ¿Hay excepciones legales (por ejemplo, para interoperabilidad)?
    3. ¿Qué riesgos legales enfrentaría un desarrollador que la practique sin permiso?
    4. ¿La empresa menciona leyes específicas (como la DMCA o la Ley de Propiedad Intelectual)?
    5. ¿La restricción está escrita de forma clara o ambigua?
- Conclusiones claras y formales que respondan:
    1. ¿Qué diferencias existen entre las dos aplicaciones?
    2. ¿Qué postura tienen las empresas frente a la ingeniería inversa?
    3. ¿Estas restricciones afectan la innovación, la educación o la seguridad?
    4. ¿Qué deberían saber los futuros ingenieros sobre este tema?
- Referencias en formato IEEE.

El texto debe ser formal, coherente, sin repetir ideas, sin usar dos puntos al inicio de párrafos y con lenguaje claro y entendible.
